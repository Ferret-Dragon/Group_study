Question 3: Reduction Logic (20 points)

Given: A reduces to B in polynomial time (A ≤p B)

Key principle: If A ≤p B, then solving B solves A. Therefore:

  • If B is easy, then A must be easy (because we can solve A using B)
  • If A is hard, then B must be hard (because making B easy would make A easy)
If this is true Then you can infer:
We know A must be slow (hard) B must also be slow (hard)
Reason: If B were fast, we could solve A in polynomial time (reduction + B's time), contradicting that A is slow.
We know A is fast (easy) Nothing
Reason: B could be either fast or slow. A being easy doesn't tell us anything about B's difficulty.
We know B must be slow (hard) Nothing
Reason: A could be either easy or hard. B being hard doesn't constrain A.
We know B is fast (easy) A must also be fast (easy)
Reason: We can solve A by reducing to B (polynomial time) then solving B (fast), so total time is fast.

Memory aid: Information flows "upstream" against the reduction arrow direction. If you know something about A (the source), it tells you about B (the target) if A is hard. If you know about B, it tells you about A if B is easy.

  • "X solvable in polynomial time via nondeterminism"
  • "Given a solution, can verify it's correct in polynomial time"
Question 8: Why P ⊆ NP?

ANSWER: If you can solve a problem in polynomial time, you can verify solutions in polynomial time.

Alternative acceptable answers (≤20 words):

  • "Solving is at least as hard as verifying"
  • "Any polynomial-time algorithm can also verify in polynomial time"
  • "If solvable fast, then verifiable fast"

Detailed reasoning: To verify a proposed solution, just solve the problem yourself and compare. Since solving takes polynomial time (because it's in P), verification also takes polynomial time, so it's in NP.

Question 10: Proving NP-Completeness (10 points)

Correct answers to check:
  • ✅ Show that problem X is in NP (or equivalently...)
  • ✅ Show that problem X can be verified in polynomial time
  • ✅ Reduce SAT (or another NP-complete problem) to problem X

Incorrect answers (do NOT check):

  • ❌ Show that problem X is not in NP-easy (this isn't a standard complexity class)
  • ❌ Show that problem X can be solved in polynomial time (this would put it in P, not prove NP-completeness!)
  • ❌ Show that problem X can only be solved in exponential time (we don't know this for NP-complete problems; it's the P≠NP question)
  • ❌ Show that problem X can only be verified in exponential time (contradicts being in NP)
  • ❌ Reduce problem X to SAT (direction is backwards)
  • ❌ Show that problem X is in P (contradicts NP-completeness unless P=NP)

Complete answer format (30 words max):

"Show X is in NP by demonstrating polynomial-time verification. Reduce a known NP-complete problem to X in polynomial time." (20 words)

The two key steps:

  1. X ∈ NP: Show solutions can be verified in polynomial time
  2. X is NP-hard: Show that some known NP-complete problem Y reduces to X (Y ≤p X)
